Ubuntu 设置 python3 为默认

sudo update-alternatives --install /usr/bin/python python /usr/bin/python2 100

sudo update-alternatives --install /usr/bin/python python /usr/bin/python3 150

如果要切换回 python2

sudo update-alternatives --config python

根据提示输入选择数字回车即可。

Ubuntu 开机显示您的网络有 .local 域

用你喜欢的编辑器（vim 或 gedit等）将 /etc/default/avahi-daemon 文件中的参数改为 AVAHI_DAEMON_DETECT_LOCAL=0 即可。（原来等于 1）

Ubuntu 优化加速

使用 man systemd-analyze 查看开机启动项
发现禁用占用时间最长的 NetworkManager-wait-online.service
使用命令 sudo systemctl disable NetworkManager-wait-online.service 禁止启动
设置 vm.swappiness=10
1.打开终端，查看交换参数的值大小，输入命令:
cat /proc/sys/vm/swappiness
使用下列命令可临时将交换参数改为10，重启之后仍会恢复原状，变为60:
sudo sysctl vm.swappiness=10
永久的改变该参数，用下列命令:
gksudo gedit /etc/sysctl.conf
搜索关键字 “vm.swappiness”（无引号），改成你愿意的值。如果 vm.swappiness 不存在，可在文件末尾添加:
vm.swappiness=10，完毕，保存文件，重启即可。

Preload 是一个运行于后台的监护程序，探测那些常用的软件，并将其放入缓存，以起到加速的作用。
在LinuxMint/Ubuntu下安装Preload很简单：
sudo apt-get install preload
Preload 默认的配置对于普通用户而言已经不错了，一般不需要修改。如果有进一步掌控其的欲望，可以打开其配置文件进行修改：
sudo gedit /etc/preload.conf

Microsoft Word 2016 无法正常撤销

电脑安装 Word 2016 后，有时候会遇到撤销功能无法正常使用，无论是使用上方的按钮还是使用 ctrl+z 快捷键都不行，按钮始终是灰色的，或者点一下就变灰了。
去网上查了查，主要有以上几种解决办法

  • 重装office
  • 更换输入法（Microsoft 官方推荐，然而好像并不管用）
  • 修改注册表（网友说有事时好时坏）

我是用第三种方法解决的。

  1. win+R 输入 regedit 进入注册表编辑器
  2. 找到 HKEY_CURRENT_USER/Software/Microsoft/Office/15.0/Word/Options
  3. 新建 DWORD 值，名为 UndoHistory，双击修改它的值为 十进制 0~100 的数（我填的100）

是没有办法隐藏标题栏的。

  • 在 setContentView 方法后添加一句:
    getSupportActionBar().hide();

Android Studio 导入项目的正确步骤

  1. 打开 Project 中的 build.gradle, 修改 dependicies 为自己本地的版本
  2. 打开 Project 中 gradle-wrapper.properties, 将 distributionUrl修改为自己本地的版本
  3. 打开 app 中的 build.gradle, 修改 compileSdkVersion 和 buildToolsVersion 为自己本地的版本

Ubuntu 下 Sublime Text 3 无法输入中文

保存下面的代码到文件sublime_imfix.c(位于~目录)

将上一步的代码编译成共享库libsublime-imfix.so，命令
gcc -shared -o libsublime-imfix.so sublime_imfix.c pkg-config --libs --cflags gtk+-2.0 -fPIC
如果提示 gtk/gtkimcontext.h：没有那个文件或目录，那就是没有相关的依赖软件，安装命令：
sudo apt-get install build-essential libgtk2.0-dev
然后将libsublime-imfix.so拷贝到sublime_text所在文件夹
sudo mv libsublime-imfix.so /opt/sublime_text/
修改文件/usr/bin/subl的内容
sudo gedit /usr/bin/subl
将
#!/bin/sh
exec /opt/sublime_text/sublime_text "$@"
修改为
#!/bin/sh
LD_PRELOAD=/opt/sublime_text/libsublime-imfix.so exec /opt/sublime_text/sublime_text "$@"
此时，在命令中执行 subl 将可以使用搜狗for linux的中文输入
为了使用鼠标右键打开文件时能够使用中文输入，还需要修改文件 sublime_text.desktop 的内容。
sudo gedit /usr/share/applications/sublime_text.desktop
将[Desktop Entry]中的字符串
Exec=/opt/sublime_text/sublime_text %F
修改为
Exec=bash -c "LD_PRELOAD=/opt/sublime_text/libsublime-imfix.so exec /opt/sublime_text/sublime_text %F"
将[Desktop Action Window]中的字符串
Exec=/opt/sublime_text/sublime_text -n
修改为
Exec=bash -c "LD_PRELOAD=/opt/sublime_text/libsublime-imfix.so exec /opt/sublime_text/sublime_text -n"
将[Desktop Action Document]中的字符串
Exec=/opt/sublime_text/sublime_text --command new_file
修改为
Exec=bash -c "LD_PRELOAD=/opt/sublime_text/libsublime-imfix.so exec /opt/sublime_text/sublime_text --command new_file"
注意：修改时请注意双引号"",否则会导致不能打开带有空格文件名的文件。
此处仅修改了 /usr/share/applications/sublime-text.desktop，但可以正常使用了。/opt/sublime_text/目录下的 sublime-text.desktop 可以修改，也可不修改。
