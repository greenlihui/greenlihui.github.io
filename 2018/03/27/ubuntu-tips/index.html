Ubuntu16.04 个性化

today 2018-03-27 visibility 0 local_offer #tips #Ubuntu

Last Updated on 2018/04/09

这是安装完 ubuntu16.04 后，自己的一些个性化，还有一些我个人需要安装的软件。因为是写给自己用的，所以有些地方没有写那么仔细。

删除自带软件

  1. 删除 libreoffice
    sudo apt remove libreoffice?
  2. 删除 amazon 链接
    sudo apt remove unity-webapps-common
  3. 删除其他自带软件
    sudo apt-get remove thunderbird totem rhythmbox empathy brasero simple-scan gnome-mahjongg aisleriot gnome-mines cheese transmission-common gnome-orca webbrowser-app gnome-sudoku landscape-client-ui-install onboard deja-dup

命令行安装一些程序

  • 安装 vim
    sudo apt install vim
  • 安装 git
    sudo apt install git，安装完记得配置
  • 安装压缩和解压
    1. 压缩功能
      安装 sudo apt-get install rar
      卸载 sudo apt-get remove rar
    2. 解压功能
      安装 sudo apt-get install unrar
      卸载 sudo apt-get remove unrar
    3. 使用压缩和解压缩.rar
      解压：rar x FileName.rar
      压缩：rar a FileName.rar DirName

安装 zsh 和 oh-my-zsh

  • 安装 zsh
    sudo apt install zsh
  • 下载 oh-my-zsh 来配置 zsh
    wget https://github.com/robbyrussell/oh-my-zsh/raw/master/tools/install.sh -O - | sh
    或者
    sh -c "$(curl -fsSL https://raw.githubusercontent.com/robbyrussell/oh-my-zsh/master/tools/install.sh)"
  • 设置 zsh 为默认 shell
    chsh -s /bin/zsh

  • 安装配套图标
  • 使用 unity-tweak-tool 修改主题和图标

关于提示 “当前网络有local域”

修改/etc/default/avahi-daemon 中参数
AVAHI_DAEMON_DETECT_LOCAL=0

“此软件来自第三方且可能包含非自由组件” 的问题

Ubuntu16.04 LTS安装第三方软件 *.deb 时， 提示此软件来自第三方且可能包含非自由组件的在Ubuntu软件中心安装GDebi，然后右键单击 *.deb 安装包，打开方式选择”GDebi包安装程序”，最后在机器上 sudo apt install *.deb 就ok

火狐浏览器设置对所有页面缩放

  1. 地址栏输入 about:config
  2. 找 layout.css.devPixelsPerPs
  3. 修改数值 1.25，意思就是 125%

设置外置磁盘开机自动挂载

  1. sudo blkid 查看磁盘分区 UUID
  2. vim /etc/fstab i 进入编辑模式
  3. 最后一行添加 UUID=xxxxx 挂载点 ext4 auto 0 0

设置启动器位置

启动器位于下方
gsettings set com.canonical.Unity.Launcher launcher-position Bottom
启动器位于左侧
gsettings set com.canonical.Unity.Launcher launcher-position Left

禁用客人会话

sudo gedit /etc/lightdm/lightdm.conf
添加如下代码，然后保持重启电脑即可。

ibus 拼音输入法

安装：sudo apt-get install ibus-pinyin
无法正常输入：ibus-daemon -drx

安装 WPS 后字体缺失

  1. 下载：wps_symbol_fonts.zip
  2. 创建目录：sudo mkdir /usr/share/fonts/wps-office
  3. 将下载的字体复制到创建的目录：sudo cp -r wps_symbol_fonts.zip /usr/share/fonts/wps-office
  4. 解压字体包：sudo unzip wps_symbol_fonts.zip
  5. 解压后删除字体包：sudo rm -r wps_symbol_fonts.zip
    ps:安装 wps 去英文官网下载（更新更快）

AS 报错 unable to sun mksdcard

Android Studio 报错 unable to run mksdcard sdk tool
打开终端输入sudo apt-get install lib32stdc++6

离线安装包安装和配置JDK

安装 tomcat

  1. 下载tar.gz安装包，解压到想安装的位置开始配置
  2. vim /etc/profile 添加
  3. export CATALINA_HOME=解压的位置
  4. 进入tomcat bin目录
  5. vim catalina.sh
  6. 找到这行# OS specific support. $var must be set to either true or false.
  7. 在这行下面新增如下配置语句
    CATALINA_HOME=...
    JAVA_HOME=...
  8. 安装tomcat服务
  9. cp catalina.sh /etc/init.d/tomcat
  10. 启动和关闭tomcat
  11. service tomcat start（stop）

安装 MySql

  1. sudo apt-get install mysql-server mysql-client
  2. Eclipse添加tomcat时显示 tomcat unknown version of tomcat was specified
  3. 因为配置tomcat需要访问tomcat目录下的lib库。
    通过如下命令：
    chmod -R 777 apache-tomcat-xxx/lib
    Eclipse无法新建服务器时需要获得tomcat目录执行权限
    sudo chmod 777 tomcatdir -R

添加快捷方式（Intellij-IDEA为例）

  • sudo vim /usr/share/applications/intellij-idea.desktop
  • sudo chmod +x /usr/share/applications/intellij-idea.desktop

eclipse 创建 AVD 模拟器报错 failed to create the sd card

两种情况：

  1. 设置android sdk下的tools权限为可执行权限。Chmod +x tools目录
  2. 没有安装ia32-libs
    1)，在用的架构:
    dpkg --print-architecture
    输出当前的使用的架构，比如amd64
    2)，添加架构
    dpkg --add-architecture i386
    添加i386架构，然后刷新下源列表，当然，删除就是remove
    apt-get update
    这时就已经包含i386的软件列表
    3)，安装i386软件
    apt-get install iceweasel:i386
